# PPTX绘图风格完整指南 v2

> 基于某41页学术项目申报PPT图集全部41页的逐页渲染视觉分析 + OOXML形状级数据提取，
> 由41个并行视觉分析与3路综合归纳生成（2026-07）。
> 本指南取代v1。v1的核心错误：把主题基色当作卡片填充色（实际渲染带60-80%浅化tint），
> 误以为深色填充+白字是主导模式（实际主导是"浅色填充+黑1.5pt边+黑字"）。

## 风格签名（一句话）

**浅色tint填充 + 黑色1.5pt实线边框 + 黑色粗体标题/常规正文**的高密度卡片体系；
sysDash虚线分组框配"骑缝标签"；块状实心箭头（金黄FFC000+黑描边为签名）做宏观流程，
细线连接器只做微观汇聚；一区一色、同色相三档深浅递进；白字只出现在饱和深色窄条上。

## 1. 配色系统

## 2. 密度规则

- **形状数量区间**：16–110个/页，均值≈50。分三档：疏朗页16-35（总览/单概念图）、标准页40-65（技术路线）、高密页70-110（总体架构）。
- **占版率**：主流60–95%，均值≈70%。总览类构图页可低至40-55%（内容居中，四周大留白）。
- **边距**：左右常见35–140px（1280宽坐标系），最紧35px、最松310px；顶部留50–100px给页标题；**底部是标配留白区**，常见75–180px（放图注或纯空）。
- **元素间距模数**：卡片间距8–15px；分组框内边距10–25px；层/栏间隙25–50px（正好容纳一枚块箭头）；栏间距10–25px。
- **何时留白**：① 单一概念/循环图（<30形状）→ 居中构图、四周留白各>130px；② 页面底部永远优先留白而非顶部；③ 高密页（>70形状）靠"栏容器+统一卡片模数+三级同色嵌套"维持秩序，不靠留白。
- **对齐纪律**：同级卡片严格等宽等高、y坐标对齐；三栏底部输出节点强制基线对齐；表头箭头与下方泳道框严格等宽。

## 3. 连接语义

| 连接件 | 语义 | 画法参数 |
|---|---|---|
| **黄色块箭头** | 层级支撑主线（下层支撑上层）、主干流程步进 | upArrow/rightArrow预设形，fill `#FFC000`（或grad FFC000→FFC000），**黑描边1–1.5pt**，尺寸26–54px；旁配无框透明文本框写关系词（"支撑""能力封装"，11–14pt） |
| **蓝色块箭头** | 模块间数据流/阶段递进 | fill `#4472C4` 实心，黑1pt描边或无描边，约30–58px；**方向靠rot旋转实现**：rightArrow+rot=5400000(向下)/16200000(向上)/10800000(向左)，很少直接用downArrow/leftArrow |
| **白底空心箭头** | 跨模块/跨课题数据流（清淡不抢眼） | rightArrow 23×17px，fill `#FFFFFF` + border `#4472C4`/1.5pt |
| **灰色块箭头** | 次级步骤衔接、低权重流程、时间轴 | fill `#A5A5A5`/`#E7E6E6`/`#9CA3AF`，13–21px小尺寸，深灰或同色描边 |
| **细线连接器** | **多对一汇聚/一对多扇出、卡片级逻辑流、拓扑节点关系**——从不做主干 | straightConnector1/bentConnector3，**1.5pt**，端头实心小三角；颜色跟随所在分区accent（蓝5B9BD5/绿70AD47/橙ED7D31/黄FFC000）或纯黑；双向关系用两根对置短线拼成 |
| **虚线** | **只表分组/分区，从不表流向**：sysDash 1.5pt roundRect=逻辑分组框（黑或主题色边，常配"骑缝标签"压其上边）；sysDot 2pt=accent色分组框；dash直线=矩阵分区分隔线 | fill:none或极浅同系底色 |
| **弧形箭头** | 循环/反馈回路 | curvedRightArrow/curvedDownArrow，蓝实心，可拉到88×415超高比例做整页回环 |
| **花括号 brace** | 汇聚归纳（替代箭头） | rightBrace/leftBrace，1.5–2.25pt，颜色=所在面板主题色加深一档 |
| **homePlate+chevron** | 页首阶段导航（首段homePlate、后续chevron，x重叠4-5px咬合），本身即块箭头 | h≈40–81px，饱和填充，白粗体16–20pt字（常为叠加透明文本框） |
| **mathPlus/plus** | "A+B组合"关系 | 实心预设形状，21–48px，夹在并列元素间 |

**两套箭头分工铁律**：块状箭头=宏观流程/支撑/递进；细线=微观汇聚/内容级数据流。全页可只用其一，但语义不混用。

## 4. 文字规则

- **字号层级**（标准页）：
  - 页标题：24–28pt 黑粗体，左上角
  - 总标题条/页签：20pt **白**粗体（饱和深底上）
  - 分区/栏/面板标题：16–18pt 黑粗体
  - 一级节点/组标题/侧标签：14–16pt 粗体
  - 内容卡正文：12–14pt **常规**
  - 子模块小卡：10–11pt 常规
  - 箭头旁注/脚注：11–12pt 常规（高密页整体下移至9–10.5pt）
- **粗体使用率**：约50–60%。规则明确：**标题/标签/节点层100%粗体，内容卡/子卡正文100%常规**——粗体即层级标记，不做局部强调。
- **黑字条件**（默认，≈85%）：所有pastel浅色填充上、所有无框文本、虚线框内正文。注意：**中饱和色卡（83CBEB/FFC000/ED7D31标题条）也配黑粗体字**，不用白字。
- **白字条件**（仅限）：饱和深色填充上——深蓝标题条（0070C0/4472C4/305598）、chevron/homePlate页签、深色一级节点与结论条（ED7D31/70AD47原色）、序号徽章。
- **红字/彩字条件**（极少）：① 竖排侧柱上的红色`#FF0000`关键词标签；② 冲突/告警标注 `#EE0000`（红×线、⚠符号）配绿`#70AD47`✓做"结论着色"混色run；③ 指标卡内数字用`#ED7D31`橙高亮（黑正文+橙数字混排）。
- **竖排手法**：多用窄宽roundRect（35–70px宽）强制逐字换行的"伪竖排"，少数用vert=eaVert；侧标签是粗体。
- **标注文字**：箭头/连线旁的关系词一律放**无填充无边框透明文本框**，不写在箭头形状内。

## 5. 反面清单（生成器必须避免）

1. **大面积饱和深色填充+白字做内容区** —— 饱和色只出现在窄标题条、页签、结论条、小徽章（高度≤80px的条状元素）；面板/大容器永远是近白浅色。
2. **无边框的浅色卡片** —— 浅色卡必带黑（偶深蓝）1–1.5pt实边；无边框仅允许出现在饱和色标题条/胶囊和透明文本框上。
3. **细线箭头做主干流程** —— 主流程/层级支撑永远用块状实心箭头；细线只承担汇聚、扇出、卡内逻辑。
4. **虚线箭头/虚线表流向** —— 虚线专属分组分区语义。
5. **深色/彩色页面背景** —— 全部白底；也没有全页渐变背景。
6. **投影、发光、立体、多色渐变** —— 唯一渐变是箭头的FFC000→FFC000同色渐变；唯一3D是个别圆柱/圆盘平台。
7. **灰色正文文字** —— 文字近乎纯黑（Tailwind灰阶仅1页特例），不用#808080级说明文字。
8. **空心/描边式内容卡（fill:none白卡漂浮）** —— 无填充+边框只用于最外层画布容器和虚线分组框。
9. **超过4个色相同页混用** —— 每页1–4个色系，且严格"一区一色/一栏一色"编码；不出现彩虹式配色。
10. **大圆角/胶囊形卡片** —— roundRect圆角一律很小；真正的胶囊形不出现。
11. **粗体做行内局部强调** —— 强调靠颜色（红/橙数字）或独立标签，不靠正文局部加粗。
12. **箭头内写文字** —— 关系词永远在旁侧透明文本框。
13. **>28pt的超大字、艺术字、字体阴影** —— 不出现。
14. **直接使用leftArrow/downArrow预设** —— 该风格惯用rightArrow/upArrow+rot旋转（5400000/10800000/16200000）实现其他方向（复现应保持此惯例以统一箭头头部比例）。
15. **饱和主题原色直接做卡片填充**（如纯4472C4/ED7D31大卡+黑字）—— 卡片必须浅化至pastel档；原色只留给标题条、箭头和accent。
16. **无标题条的分组框** —— 虚线分组框几乎总配"骑缝/压框标签"（饱和色标题条骑压框边线15–40px），不出现光秃虚线框。

# PPTX 绘图风格组件词汇表

> 归纳自41页逐页分析。整套PPT的核心签名是：**"浅色tint填充 + 黑色1.5pt实线边框 + 黑色粗体/常规字"** 的高对比统一卡片体系，配合 sysDash 虚线分组框、骑缝标签、燕尾流程头和块状实心箭头。原文件大量使用"主题基色hex + 高透明度/lumMod浅化"，digest中记录的是基色，**复现时必须显式使用浅化后的hex**。

---

## 一、填充↔边框↔文字 配对规律（全局统计）

| 配对模式 | 占比(约) | 典型hex | 使用场景 |
|---|---|---|---|
| **浅底 + 黑边1.5pt + 黑字**（主导模式） | **≈70%** | 浅蓝#DAE3F3/#DEEBF7/#D4EBFC、浅橙#FBE5D6/#F8CBAD、浅黄#FFF2CC/#FFE699、浅紫#F5EBFF/#D8DBFC、浅绿#E2EFDA + border #000000/1.5pt + text #000000 | 所有内容卡、子模块卡、容器底板、色带、胶囊——即"信息承载层"一律浅底黑字 |
| **深饱和底 + 白粗体字**（强调模式） | **≈20%** | #0070C0/#4472C4/#406FC4/#ED7D31/#70AD47/#305598，边框为无或黑1.5pt + text #FFFFFF bold | 仅限：总标题条、燕尾流程头、一级标题节点/胶囊、结论条、序号徽章、六边形——即"标题/导航/结论层" |
| **无填充 + 黑/彩色虚线边** | ≈7% | fill:none + sysDash #000000或分区色/1.5pt | 分组框、外框容器 |
| **白底 + 彩色细边（空心）** | ≈3% | fill:#FFFFFF + border #4472C4/1.5pt | 空心块箭头、镂空小卡 |

**关键规律**：同一色相做"三档深浅递进"——大容器最浅（≈20%浓度）→ 内容卡中浅（≈40%）→ 标题条/箭头纯色（100%配白字）。黑1.5pt边框统一压住所有浅色填充；白字**只出现在纯饱和色上**，浅色上永远黑字。

## 二、组件词汇表

### 1. 浅填充深边框圆角内容卡（基础原子组件）
- **频次**：≈36/41页，每页5–20个，全PPT最高频组件
- **规格**：roundRect，圆角半径小（约4–8px）；fill 浅tint（浅蓝#DAE3F3、浅橙#FBE5D6、浅黄#FFE699、浅紫#F5EBFF等）；border **#000000 / 1.5pt 实线**（少数1pt）；文字 #000000，标题卡14–16pt粗体、内容卡10.5–14pt常规，居中；典型尺寸 **110–280 × 30–55px**；同列等宽等高，垂直间距8–15px
- **语义**：一切研究内容/技术要点/子模块的信息载体；粗体=节点标题，常规=要素条目

### 2. 深色标题条 / 总标题胶囊
- **频次**：≈20页
- **规格**：roundRect；fill 饱和深蓝 **#0070C0 / #406FC4 / #4472C4**（分区版用#ED7D31/#5B9BD5）；无边框或黑1.5pt；文字 **#FFFFFF 粗体 16–20pt** 居中；典型 **570–850 × 40–60px**，水平居中压在容器顶部内侧
- **语义**：整图或分区的总标题；是"深底白字"的首要出现场所

### 3. 虚线分组框（sysDash）
- **频次**：≈32/41页，出现1–5个/页
- **规格**：roundRect或rect；border **1.5pt dash=sysDash**，颜色默认#000000，随分区可换（#4472C4/#ED7D31/#0070C0，个别页用sysDot 2pt）；fill 三种变体：①无填充 ②白色 ③本区浅tint底板（如#FBE5D6/#EEEDFD）；典型 **215–890 × 90–460px**，内边距10–25px
- **语义**：逻辑分组/阶段聚合（虚线=逻辑边界，实线=实体卡片）；框内套2×2或纵向小卡阵列

### 4. 骑缝标签（压框标题条）★标志性手法
- **频次**：≈12页（slides 2,3,6,7,8,13,17,18,19,21,36,38）
- **规格**：roundRect实心饱和填充（#0070C0/#ED7D31/#FFC000/#7030A0等），无边框；文字白粗体16pt（深底）或黑粗体16pt（黄/浅底）；**垂直骑跨在虚线分组框上边线上，重叠15–40px**；典型 **180–320 × 37–80px**，水平居中
- **语义**：分组标题，用层叠制造标题与内容的从属关系，全套PPT最具辨识度的手法

### 5. 燕尾流程条（homePlate + chevron 表头）
- **频次**：11页（5,6,10,13,14,17,18,19,21,36,38）
- **规格**：**首段用homePlate（单尖五边形），后续段用chevron（燕尾）**，相邻段x重叠4–9px咬合；高 **40–81px**，每段宽300–530px，与下方栏面板严格等宽对齐；fill饱和色无边框（或黑1.5pt边），常见两种配色策略：①同色相深→浅梯度（#09467D→#0070C0→#00B0F0）②每段一色对应栏色（蓝#0070C0/橙#ED7D31/绿#70AD47）；文字 **白粗体16–20pt**（常用独立透明文本框叠放）
- **语义**：页顶阶段导航/"表头即列头"，形状本身承担流程方向语义，段色=下方泳道色

### 6. 黑边外框容器 / 全高栏面板
- **频次**：≈28页

## 三、复现要点速记

1. **tint法则**：digest/XML记录主题基色（#4472C4/#ED7D31/#FFC000/#5B9BD5/#70AD47），渲染均为60–80%浅化。复现时容器用≈20%浓度、内容卡≈40%、标题/箭头用100%基色，**边框和文字保持纯黑不浅化**。
2. **旋转惯例**：向上/向下/向左箭头一律用 rightArrow/upArrow + rot（5400000/10800000/16200000），不换预设形状。
3. **线宽体系**：几乎唯一线宽 **1.5pt**（内层小卡1pt，极细边0.5–0.75pt仅见于Tailwind风格页17）。
4. **虚线**：分组一律 **sysDash**（少数sysDot 2pt），虚线=逻辑分组，实线=实体。
5. **一区一色**：每栏/每层一个色相，栏内做"流程头深→面板近白→卡片中浅→箭头/括号/连线accent"四层同族配色；连接线、花括号颜色跟随所在分区而非全局黑。
6. **字号阶梯**：页标题28pt > 总标题条20pt > 分组标题14–16pt粗体 > 内容卡14pt常规 > 小卡10.5–12pt > 旁注11–12pt。
